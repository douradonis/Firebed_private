Email Configuration
Select the email service to use for sending emails.
SMTP: Traditional email using SMTP server (configured via environment variables)
Resend: Modern API-based email service (requires RESEND_API_KEY in .env)
OAuth2 Outlook: Microsoft Outlook with OAuth2 authentication
Mailpit: Email testing tool for development (captures emails locally, requires Mailpit server running)
Email Configuration Guide
SMTP Configuration

Set these environment variables in your .env file:

SMTP_SERVER=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SENDER_EMAIL=your-email@gmail.com
Resend Configuration

Set this environment variable in your .env file:

RESEND_API_KEY=re_xxxxxxxxxxxxx
SENDER_EMAIL=your-verified-sender@yourdomain.com

Note: You need to verify your sending domain in Resend dashboard first.

Mailpit Configuration (Development/Testing)

Mailpit is an email testing tool that captures emails locally instead of sending them.

Installation:

  • Mac: brew install mailpit or brew services start mailpit
  • Docker: docker run -d --name mailpit -p 8025:8025 -p 1025:1025 axllent/mailpit
  • Linux/Windows: Download from GitHub releases

Set these environment variables in your .env file:

MAILPIT_API_URL=http://localhost:8025
SENDER_EMAIL=noreply@test.local

Optional authentication:

MAILPIT_API_USERNAME=your-username
MAILPIT_API_PASSWORD=your-password

Access: After starting Mailpit, view captured emails at http://localhost:8025

Note: Emails are NOT actually sent - they are captured for testing only.
